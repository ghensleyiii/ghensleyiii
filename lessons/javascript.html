Lesson 1: What is JavaScript?

Introduction to JavaScript

JavaScript is a versatile, high-level programming language used primarily for web development. It adds interactivity and dynamic behavior to websites, running in browsers or on servers via Node.js. Learning JavaScript is essential for modern web applications.

console.log("Hello, World!");

Output: Hello, World!

Lesson 2: Variables

Variables in JavaScript

Variables store data using var, let, or const keywords. Let and const, introduced in ES6, provide block scope, with const for unchangeable values. Variables are fundamental for holding and manipulating data.

let name = "Alice";
const age = 25;
console.log(name, age);

Output: Alice 25

Lesson 3: Data Types

JavaScript Data Types

JavaScript has dynamic types, including numbers, strings, booleans, objects, and undefined. The typeof operator identifies a value’s type. Understanding types is key to effective programming.

let num = 42;
let str = "Hello";
console.log(typeof num, typeof str);

Output: number string

Lesson 4: Arithmetic Operators

Arithmetic Operations

JavaScript supports operators like +, -, *, /, and % for calculations. The ** operator performs exponentiation. These operators are essential for numerical computations.

let a = 10;
let b = 3;
console.log(a + b, a * b, a % b);

Output: 13 30 1

Lesson 5: Comparison Operators

Comparison Operators

Comparison operators like ==, ===, !=, and > return booleans. The === operator checks value and type, avoiding type coercion. Strict equality (===) is preferred for reliable comparisons.

let x = 5;
let y = "5";
console.log(x == y, x === y);

Output: true false

Lesson 6: Logical Operators

Logical Operators

Logical operators &&, ||, and ! combine or invert boolean expressions. They’re used for conditional logic and short-circuit evaluation. Logical operators are crucial for complex conditions.

let a = true;
let b = false;
console.log(a && b, a || b, !a);

Output: false true false

Lesson 7: Strings

Working with Strings

Strings are sequences of characters, created with single or double quotes. They support concatenation with + and template literals with backticks. Strings are fundamental for text manipulation.

let greeting = "Hello";
let name = "World";
console.log(`${greeting}, ${name}!`);

Output: Hello, World!

Lesson 8: String Methods

String Methods

JavaScript strings have methods like toUpperCase(), trim(), and slice() for manipulation. These methods return new strings, leaving the original unchanged. String methods simplify text processing tasks.

let text = "  JavaScript  ";
console.log(text.trim().toUpperCase());

Output: JAVASCRIPT

Lesson 9: Arrays

Working with Arrays

Arrays are ordered lists created with square brackets. They support methods like push() and pop() for manipulation. Arrays are versatile for storing multiple values.

let fruits = ["apple", "banana"];
fruits.push("cherry");
console.log(fruits);

Output: ["apple", "banana", "cherry"]

Lesson 10: Array Methods

Array Methods

Array methods like map(), filter(), and sort() transform or query arrays. They’re functional and don’t modify the original array (except sort()). These methods streamline data processing.

let numbers = [1, 2, 3];
let squares = numbers.map(n => n * n);
console.log(squares);

Output: [1, 4, 9]

Lesson 11: If Statements

Conditional Statements

If statements control program flow with if, else if, and else. They execute code based on boolean conditions. Conditionals are essential for decision-making logic.

let age = 18;
if (age >= 18) {
    console.log("Adult");
} else {
    console.log("Minor");
}

Output: Adult

Lesson 12: Switch Statements

Switch Statements

Switch statements evaluate an expression and execute matching case blocks. They’re an alternative to multiple if-else statements for discrete values. Use break to prevent fall-through.

let day = 1;
switch (day) {
    case 1:
        console.log("Monday");
        break;
    default:
        console.log("Other");
}

Output: Monday

Lesson 13: For Loops

For Loops

For loops iterate over a range or array with a counter variable. They’re ideal for known iteration counts. Loops are fundamental for repetitive tasks.

for (let i = 0; i < 3; i++) {
    console.log(i);
}

Output:
0
1
2

Lesson 14: While Loops

While Loops

While loops run as long as a condition is true. They’re useful when the number of iterations is unknown. Use them carefully to avoid infinite loops.

let count = 1;
while (count <= 3) {
    console.log(count);
    count++;
}

Output:
1
2
3

Lesson 15: Break and Continue

Loop Control

The break statement exits a loop, while continue skips to the next iteration. They provide precise control over loop behavior. These statements handle special cases efficiently.

for (let i = 0; i < 5; i++) {
    if (i === 3) break;
    console.log(i);
}

Output:
0
1
2

Lesson 16: Functions

Defining Functions

Functions encapsulate reusable code, declared with the function keyword. They accept parameters and return values. Functions promote modularity and code organization.

function greet(name) {
    return `Hello, ${name}!`;
}
console.log(greet("Alice"));

Output: Hello, Alice!

Lesson 17: Function Expressions

Function Expressions

Function expressions assign functions to variables, allowing anonymous functions. They’re useful for callbacks and immediate invocation. Function expressions are flexible for dynamic code.

const add = function(a, b) {
    return a + b;
};
console.log(add(2, 3));

Output: 5

Lesson 18: Default Parameters

Default Parameters

Default parameters provide fallback values for missing arguments. They simplify function calls and enhance flexibility. Defaults are common in modern JavaScript functions.

function greet(name = "Guest") {
    return `Hello, ${name}!`;
}
console.log(greet());

Output: Hello, Guest!

Lesson 19: Objects

Working with Objects

Objects store key-value pairs, created with curly braces. They support methods and dynamic property access. Objects are central to JavaScript’s data structures.

let person = {
    name: "Bob",
    age: 30
};
console.log(person.name);

Output: Bob

Lesson 20: Object Methods

Object Methods

Object methods are functions stored as properties. The this keyword refers to the object in methods. Methods enable objects to encapsulate behavior.

let person = {
    name: "Alice",
    greet() {
        return `Hi, I'm ${this.name}`;
    }
};
console.log(person.greet());

Output: Hi, I'm Alice

Lesson 21: DOM Selection

DOM Manipulation

The Document Object Model (DOM) represents a webpage’s structure. Methods like querySelector() select elements for manipulation. DOM access is key to interactive websites.

let element = document.querySelector("h1");
console.log(element.textContent);

Output: (Text of first h1 element, e.g., "CODEXEDOC")

Lesson 22: DOM Modification

Modifying the DOM

DOM properties like textContent and style modify element content and appearance. These changes create dynamic user interfaces. DOM modification is core to web interactivity.

let element = document.querySelector("p");
element.textContent = "Updated text";

Output: (Changes paragraph text to "Updated text")

Lesson 23: Event Listeners

Handling Events

Event listeners, added with addEventListener(), respond to user actions like clicks. They enable interactive features like buttons. Events drive modern web applications.

let button = document.querySelector("button");
button.addEventListener("click", () => {
    console.log("Clicked!");
});

Output: Clicked! (on button click)

Lesson 24: Event Object

Event Object

The event object, passed to event handlers, contains details like the target element. Properties like event.target access the clicked element. The event object enhances event-driven programming.

document.addEventListener("click", (event) => {
    console.log(event.target);
});

Output: (Logs the clicked element, e.g.,

Lesson 25: Arrow Functions

Arrow Functions

Arrow functions, introduced in ES6, provide concise syntax with =>. They don’t bind their own this, making them ideal for callbacks. Arrow functions simplify functional programming.

const add = (a, b) => a + b;
console.log(add(2, 3));

Output: 5

Lesson 26: Template Literals

Template Literals

Template literals, enclosed in backticks, allow embedded expressions with ${}. They support multiline strings and interpolation. Template literals improve string formatting readability.

let name = "Bob";
let greeting = `Hello, ${name}!`;
console.log(greeting);

Output: Hello, Bob!

Lesson 27: Destructuring

Destructuring Assignment

Destructuring extracts values from arrays or objects into variables. It simplifies code by reducing repetitive access. Destructuring is a powerful ES6 feature.

let [a, b] = [1, 2];
console.log(a, b);

Output: 1 2

Lesson 28: Spread Operator

Spread Operator

The spread operator (...) expands arrays or objects into individual elements. It’s used for copying, merging, or passing arguments. Spread simplifies array and object manipulation.

let arr1 = [1, 2];
let arr2 = [...arr1, 3];
console.log(arr2);

Output: [1, 2, 3]

Lesson 29: Rest Parameters

Rest Parameters

Rest parameters, using ..., collect remaining arguments into an array. They’re useful for functions with variable argument counts. Rest parameters enhance function flexibility.

function sum(...numbers) {
    return numbers.reduce((a, b) => a + b, 0);
}
console.log(sum(1, 2, 3));

Output: 6

Lesson 30: Promises

Promises

Promises represent asynchronous operations, with states like pending, fulfilled, or rejected. They use .then() and .catch() for handling results. Promises simplify asynchronous code management.

let promise = new Promise((resolve) => {
    setTimeout(() => resolve("Done"), 1000);
});
promise.then(result => console.log(result));

Output: Done (after 1 second)

Lesson 31: Async/Await

Async/Await

Async functions, marked with async, return promises and use await to pause for results. They make asynchronous code read like synchronous code. Async/await improves readability for async operations.

async function fetchData() {
    let result = await Promise.resolve("Data");
    console.log(result);
}
fetchData();

Output: Data

Lesson 32: Try/Catch

Error Handling

Try/catch blocks handle errors in synchronous and asynchronous code. They prevent crashes by catching exceptions. Error handling ensures robust applications.

try {
    throw new Error("Oops");
} catch (error) {
    console.log(error.message);
}

Output: Oops

Lesson 33: Modules (Export)

Modules: Export

ES6 modules allow exporting functions, objects, or values using export. They promote code organization and reuse. Modules are standard for modern JavaScript projects.

// math.js
export function add(a, b) {
    return a + b;
}

Output: (Exports add function for import)

Lesson 34: Modules (Import)

Modules: Import

The import statement brings exported items into a file. It supports named and default imports. Importing enables modular, maintainable codebases.

// main.js
import { add } from './math.js';
console.log(add(2, 3));

Output: 5

Lesson 35: Classes

Classes

Classes, introduced in ES6, define blueprints for objects with constructors and methods. They simplify object-oriented programming. Classes are widely used for structured code.

class Person {
    constructor(name) {
        this.name = name;
    }
    greet() {
        return `Hi, ${this.name}`;
    }
}
let person = new Person("Alice");
console.log(person.greet());

Output: Hi, Alice

Lesson 36: Inheritance

Class Inheritance

Inheritance, using extends, allows a class to inherit properties and methods. The super() function calls the parent constructor. Inheritance promotes code reuse.

class Animal {
    constructor(name) {
        this.name = name;
    }
}
class Dog extends Animal {
    bark() {
        return `${this.name} barks`;
    }
}
let dog = new Dog("Max");
console.log(dog.bark());

Output: Max barks

Lesson 37: Getters and Setters

Getters and Setters

Getters and setters, defined with get and set, control property access. They enable validation and computed properties. Getters/setters enhance class encapsulation.

class Person {
    constructor(name) {
        this._name = name;
    }
    get name() {
        return this._name;
    }
    set name(value) {
        this._name = value;
    }
}
let person = new Person("Bob");
console.log(person.name);

Output: Bob

Lesson 38: Static Methods

Static Methods

Static methods, defined with static, belong to the class, not instances. They’re useful for utility functions. Static methods organize class-level functionality.

class MathUtils {
    static add(a, b) {
        return a + b;
    }
}
console.log(MathUtils.add(5, 3));

Output: 8

Lesson 39: Closures

Closures

Closures occur when a function retains access to its lexical scope’s variables. They’re used for data privacy and function factories. Closures are a powerful JavaScript feature.

function makeCounter() {
    let count = 0;
    return () => count++;
}
let counter = makeCounter();
console.log(counter());

Output: 0

Lesson 40: Prototypes

Prototypes

Prototypes are JavaScript’s mechanism for inheritance, where objects inherit properties. The prototype property adds methods to all instances. Prototypes are fundamental to JavaScript’s object model.

function Person(name) {
    this.name = name;
}
Person.prototype.greet = function() {
    return `Hi, ${this.name}`;
};
let person = new Person("Alice");
console.log(person.greet());

Output: Hi, Alice

Lesson 41: Fetch API

Fetch API

The Fetch API makes HTTP requests, returning promises. It’s used for fetching resources like JSON data. Fetch is essential for modern web APIs.

fetch("https://api.example.com/data")
    .then(response => response.json())
    .then(data => console.log(data));

Output: (Logs JSON data from API)

Lesson 42: JSON

Working with JSON

JSON (JavaScript Object Notation) is a format for data exchange. JSON.parse() and JSON.stringify() convert between strings and objects. JSON is widely used in APIs.

let jsonString = '{"name": "Alice", "age": 25}';
let obj = JSON.parse(jsonString);
console.log(obj.name);

Output: Alice

Lesson 43: Local Storage

Local Storage

LocalStorage stores key-value pairs in the browser, persisting across sessions. Methods like setItem() and getItem() manage data. LocalStorage is useful for saving user preferences.

localStorage.setItem("name", "Bob");
console.log(localStorage.getItem("name"));

Output: Bob

Lesson 44: Regular Expressions

Regular Expressions

Regular expressions (regex) match patterns in strings, created with /pattern/. Methods like test() and match() process text. Regex is powerful for validation and parsing.

let regex = /\d+/;
console.log(regex.test("123"));

Output: true

Lesson 45: SetTimeout

setTimeout

The setTimeout function delays code execution by a specified time in milliseconds. It’s used for scheduling tasks like animations. setTimeout is key for asynchronous timing.

setTimeout(() => {
    console.log("Delayed");
}, 1000);

Output: Delayed (after 1 second)

Lesson 46: SetInterval

setInterval

The setInterval function repeatedly executes code at fixed intervals. It’s used for tasks like updating a clock. Use clearInterval() to stop it.

let id = setInterval(() => {
    console.log("Tick");
}, 1000);
setTimeout(() => clearInterval(id), 3000);

Output: Tick (every second, stops after 3 seconds)

Lesson 47: Array Reduce

Array Reduce

The reduce() method accumulates array elements into a single value using a callback. It’s used for summing, flattening, or transforming data. Reduce is powerful for complex array operations.

let numbers = [1, 2, 3];
let sum = numbers.reduce((a, b) => a + b, 0);
console.log(sum);

Output: 6

Lesson 48: Map Object

Map Object

The Map object stores key-value pairs, allowing any type as keys. It provides methods like set() and get(). Maps are useful for flexible data storage.

let map = new Map();
map.set("name", "Alice");
console.log(map.get("name"));

Output: Alice

Lesson 49: Set Object

Set Object

The Set object stores unique values of any type. Methods like add() and has() manage elements. Sets are ideal for removing duplicates.

let set = new Set();
set.add(1);
set.add(1);
console.log(set.size);

Output: 1

Lesson 50: Optional Chaining

Optional Chaining

Optional chaining (?.) safely accesses nested properties, returning undefined if a property is missing. It prevents errors in deep object access. Optional chaining simplifies robust code.

let user = { profile: { name: "Alice" } };
console.log(user.profile?.name);
console.log(user.address?.city);

Output:
Alice
undefined
