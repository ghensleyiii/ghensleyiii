Lesson 1: What is HTML?

Understanding HTML

HTML, or HyperText Markup Language, is the foundation of web development, used to structure content on the internet. It consists of elements defined by tags that instruct browsers on how to display text, images, and other media. Learning HTML is the first step to creating your own websites.

<!DOCTYPE html>
<html>
<head>
    <title>My Page</title>
</head>
<body>
    <h1>Hello, World!</h1>
</body>
</html>

Hello, World!

Lesson 2: HTML Document Structure

Basic Document Structure

An HTML document begins with a DOCTYPE declaration to inform browsers of the HTML version. The <html> element contains a <head> section for metadata and a <body> section for visible content. This structure ensures proper rendering and organization of web pages.

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>My Website</title>
</head>
<body>
    <p>Welcome to my site!</p>
</body>
</html>

Welcome to my site!

Elements and Tags

HTML elements are the building blocks of web pages, defined by tags that usually come in pairs. Tags, such as <p> for paragraphs or <h1> for headings, surround content to apply structure or formatting. Understanding tags is essential for creating meaningful web content.

Lesson 11: Ordered Lists

Ordered Lists

Ordered lists, created with the <ol> tag, display items in a numbered sequence. Each item is defined using the <li> tag. They are ideal for steps, rankings, or any content requiring a specific order.

Lesson 21: Introduction to Forms

HTML Forms

Forms, created with the <form> tag, collect user input for tasks like registration or search. They include elements like <input> and <button> for data entry and submission. Forms are critical for interactive web applications.

<form action="/submit">
    <input type="text" name="username">
    <input type="submit" value="Submit">
</form>
Lesson 22: Input Types

Input Types

The <input> tag supports various types, such as text, password, and email, to suit different data needs. Each type provides specific functionality, like hiding passwords or validating emails. Choosing the right type improves user experience and data accuracy.

<input type="text" placeholder="Name">
<input type="password" placeholder="Password">
<input type="email" placeholder="Email">


Lesson 23: Labels

Form Labels

Labels, created with the <label> tag, describe the purpose of form inputs, enhancing clarity. The for attribute links a label to an input’s id, improving accessibility. Labels ensure users and screen readers understand input fields.

<input type="text" id="name">
Lesson 24: Checkboxes

Checkboxes

Checkboxes, using type="checkbox", allow users to select multiple options. Each checkbox requires a unique id and name for form submission. They are ideal for choices like selecting hobbies or preferences.

<input type="checkbox" id="option1" name="option1">
<label for="option1">Option 1</label>
<input type="checkbox" id="option2" name="option2">
<label for="option2">Option 2</label>

Lesson 25: Radio Buttons

Radio Buttons

Radio buttons, using type="radio", allow a single selection from a group, sharing the same name attribute. They are useful for exclusive choices, like selecting a gender. Each button needs a unique id for labeling.

<input type="radio" id="yes" name="choice" value="yes">
<label for="yes">Yes</label>
<input type="radio" id="no" name="choice" value="no">
<label for="no">No</label>

The <select> tag, with <option> tags, creates dropdown menus for single or multiple selections. Each option has a value attribute for form submission. Dropdowns are compact and user-friendly for choosing from predefined lists.

Form Attributes

Lesson 50: Spellcheck Attribute
