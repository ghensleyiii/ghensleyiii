Lesson 1: What is Ruby?

Introduction to Ruby

Ruby is a dynamic, object-oriented programming language known for its simplicity and developer happiness. It powers frameworks like Ruby on Rails, used for web development. Ruby’s elegant syntax makes it ideal for rapid prototyping and scripting.

puts "Hello, World!"

Output: Hello, World!

Lesson 2: Variables

Variables in Ruby

Variables in Ruby are dynamically typed, assigned without type declarations. They hold references to objects, and naming conventions indicate scope (e.g., local, instance). Ruby’s flexibility with variables simplifies coding but requires careful management.

name = "Alice"
age = 25
puts "#{name} is #{age}"

Output: Alice is 25

Lesson 3: Data Types

Ruby Data Types

Ruby supports data types like integers, floats, strings, booleans, arrays, and hashes. Everything in Ruby is an object, with methods available on all types. Understanding data types is key to leveraging Ruby’s object-oriented nature.

num = 42
pi = 3.14
flag = true
puts num, pi, flag

Output:
42
3.14
true

Lesson 4: Arithmetic Operators

Arithmetic Operations

Ruby provides arithmetic operators like `+`, `-`, `*`, `/`, and `%` for numbers. The `**` operator handles exponentiation. These operators are intuitive and essential for calculations.

a = 10
b = 3
puts a + b, a % b

Output:
13
1

Lesson 5: Comparison Operators

Comparison Operators

Ruby’s comparison operators (`==`, `!=`, `>`, `<`, `>=`, `<=`) return booleans. The `<=>` operator compares for sorting, returning -1, 0, or 1. Comparisons are vital for conditional logic.

x = 5
y = 10
puts x < y, x == y

Output:
true
false

Lesson 6: Logical Operators

Logical Operators

Ruby’s logical operators `&&`, `||`, and `!` combine or invert boolean expressions. They short-circuit for efficiency, evaluating only as needed. Logical operators are key for complex conditions.

a = true
b = false
puts a && b, a || b

Output:
false
true

Lesson 7: Strings

Working with Strings

Strings in Ruby are mutable objects, created with single or double quotes. Double-quoted strings support interpolation with `#{}`. Strings are central to text manipulation in Ruby.

greeting = "Hello"
name = "World"
puts "#{greeting}, #{name}!"

Output: Hello, World!

Lesson 8: String Methods

String Methods

Ruby’s `String` class provides methods like `upcase`, `strip`, and `gsub` for manipulation. Methods can be chained for concise operations. String methods make text processing expressive and efficient.

text = "  ruby  "
puts text.strip.upcase

Output: RUBY

Lesson 9: Arrays

Working with Arrays

Arrays in Ruby are dynamic, ordered collections, created with `[]` or `Array.new`. They support mixed types and methods like `push` and `pop`. Arrays are versatile for managing sequences.

fruits = ["apple", "banana", "cherry"]
puts fruits[1]

Output: banana

Lesson 10: Hashes

Hashes

Hashes in Ruby are key-value stores, created with `{}` or `Hash.new`. They support symbol keys (e.g., `:key`) for efficiency. Hashes are ideal for associative data.

ages = { "Alice" => 25, "Bob" => 30 }
puts ages["Alice"]

Output: 25

Lesson 11: If Statements

Conditional Statements

Ruby’s `if` statements evaluate conditions, supporting `elsif` and `else`. They can be written as modifiers (e.g., `puts x if y`). Conditionals are essential for decision-making.

age = 18
if age >= 18
  puts "Adult"
else
  puts "Minor"
end

Output: Adult

Lesson 12: Case Statements

Case Statements

Ruby’s `case` statement matches a value against patterns, similar to a switch. It supports ranges and multiple conditions with `when`. Case statements are concise for multi-way branching.

day = 1
case day
when 1
  puts "Monday"
else
  puts "Other"
end

Output: Monday

Lesson 13: Loops

Loops

Ruby provides `while`, `until`, and `loop` for iteration, though blocks are often preferred. Loops can include `break` and `next` for control. Loops are useful but less common than iterators in Ruby.

i = 0
while i < 3
  puts i
  i += 1
end

Output:
0
1
2

Lesson 14: Each Iterator

Each Iterator

The `each` method iterates over arrays or hashes, yielding elements to a block. It’s idiomatic in Ruby for collection processing. Each simplifies iteration without explicit loops.

fruits = ["apple", "banana"]
fruits.each { |fruit| puts fruit }

Output:
apple
banana

Lesson 15: Break and Next

Loop Control

The `break` keyword exits a loop or iterator, while `next` skips to the next iteration. They work within blocks and loops. These controls handle special iteration cases.

(0..4).each do |i|
  break if i == 3
  puts i
end

Output:
0
1
2

Lesson 16: Methods

Defining Methods

Methods in Ruby are defined with `def`, supporting default arguments and variable-length parameters. They return the last evaluated expression implicitly. Methods promote modularity and reusability.

def greet(name)
  "Hello, #{name}!"
end

puts greet("Alice")

Output: Hello, Alice!

Lesson 17: Blocks

Blocks

Blocks in Ruby are chunks of code passed to methods, enclosed in `{}` or `do..end`. They’re used with iterators like `each` or custom methods with `yield`. Blocks are a cornerstone of Ruby’s expressiveness.

def run_block
  yield
end

run_block { puts "Hello!" }

Output: Hello!

Lesson 18: Procs

Procs

Procs are saved blocks, created with `Proc.new` or `proc`. They can be stored and called later with `call`. Procs enable reusable code chunks in Ruby.

my_proc = Proc.new { puts "Hello!" }
my_proc.call

Output: Hello!

Lesson 19: Lambdas

Lambdas

Lambdas are similar to procs but enforce argument counts and have stricter behavior. They’re created with `lambda` or `->`. Lambdas are useful for precise function-like behavior.

my_lambda = -> { puts "Hello!" }
my_lambda.call

Output: Hello!

Lesson 20: Classes

Classes

Classes in Ruby define objects with attributes and methods, created with `class`. They support instance variables (e.g., `@var`) and methods. Classes are fundamental to Ruby’s object-oriented programming.

class Dog
  def initialize(name)
    @name = name
  end

  def bark
    "#{@name} says Woof!"
  end
end

dog = Dog.new("Buddy")
puts dog.bark

Output: Buddy says Woof!

Lesson 21: Inheritance

Inheritance

Ruby supports single inheritance with `<`, allowing subclasses to inherit methods. The `super` keyword calls the parent’s method. Inheritance organizes related classes hierarchically.

class Animal
  def speak
    "Sound"
  end
end

class Dog < Animal
  def speak
    super + ": Woof!"
  end
end

dog = Dog.new
puts dog.speak

Output: Sound: Woof!

Lesson 22: Modules

Modules

Modules in Ruby group methods and constants, used for namespacing or mixins. They’re included with `include` to add methods to classes. Modules promote code reuse and organization.

module Greetable
  def greet
    "Hello!"
  end
end

class Person
  include Greetable
end

puts Person.new.greet

Output: Hello!

Lesson 23: Error Handling

Error Handling

Ruby uses `begin`, `rescue`, and `ensure` for exception handling. Exceptions are objects, typically subclasses of `StandardError`. Error handling ensures robust programs.

begin
  raise "Error!"
rescue => e
  puts "Caught: #{e}"
end

Output: Caught: Error!

Lesson 24: Custom Exceptions

Custom Exceptions

Custom exceptions in Ruby are created by subclassing `StandardError`. They provide specific error context for better debugging. Custom exceptions enhance error handling clarity.

class MyError < StandardError; end

begin
  raise MyError, "Custom error"
rescue MyError => e
  puts e
end

Output: Custom error

Lesson 25: Symbols

Symbols

Symbols in Ruby are immutable, lightweight identifiers, prefixed with `:`. They’re often used as hash keys or method names. Symbols optimize memory usage in Ruby.

data = { name: "Alice", age: 25 }
puts data[:name]

Output: Alice

Lesson 26: Ranges

Ranges

Ranges in Ruby, created with `..` or `...`, represent sequences of values. They’re used in loops or converted to arrays. Ranges simplify working with sequences.

(1..3).each { |i| puts i }

Output:
1
2
3

Lesson 27: File Writing

File Writing

Ruby’s `File` class provides methods like `write` for file output. Blocks with `File.open` ensure files are closed properly. File writing is essential for persistent data.

File.write("output.txt", "Hello, World!")
puts "File written"

Output: File written

Lesson 28: File Reading

File Reading

The `File` class’s `read` method loads entire files into memory. It raises exceptions for errors, requiring handling. File reading is crucial for data retrieval.

content = File.read("output.txt")
puts content

Output: Hello, World!

Lesson 29: Regular Expressions

Regular Expressions

Ruby’s regular expressions, created with `/pattern/`, match and manipulate strings. Methods like `match` and `gsub` use regexes. Regexes are powerful for text processing.

text = "hello world"
puts text.gsub(/world/, "Ruby")

Output: hello Ruby

Lesson 30: Enumerable Methods

Enumerable Methods

The `Enumerable` module provides methods like `map`, `select`, and `reduce` for collections. They’re included in arrays and hashes. Enumerable methods enable functional-style programming.

numbers = [1, 2, 3]
squares = numbers.map { |n| n * n }
puts squares

Output: [1, 4, 9]

Lesson 31: Map and Collect

Map and Collect

The `map` (or `collect`) method transforms each element in a collection via a block. It returns a new array with the results. Map is widely used for data transformation.

words = ["cat", "dog"]
upper = words.map(&:upcase)
puts upper

Output: ["CAT", "DOG"]

Lesson 32: Select and Reject

Select and Reject

The `select` method filters elements where the block returns true, while `reject` keeps those returning false. Both return new arrays. These methods simplify collection filtering.

numbers = [1, 2, 3, 4]
evens = numbers.select { |n| n.even? }
puts evens

Output: [2, 4]

Lesson 33: Reduce and Inject

Reduce and Inject

The `reduce` (or `inject`) method combines elements into a single value using a block. It can take an initial value for flexibility. Reduce is useful for aggregations like sums.

numbers = [1, 2, 3]
sum = numbers.reduce(0) { |sum, n| sum + n }
puts sum

Output: 6

Lesson 34: Metaprogramming

Metaprogramming

Metaprogramming in Ruby allows dynamic code generation, using methods like `define_method`. It’s used in frameworks like Rails for flexibility. Metaprogramming enables powerful, reusable abstractions.

class Person
  define_method(:greet) do
    "Hello!"
  end
end

puts Person.new.greet

Output: Hello!

Lesson 35: Method Missing

Method Missing

The `method_missing` method handles calls to undefined methods dynamically. It’s used for delegation or custom behavior. Method missing is a key metaprogramming tool.

class Dynamic
  def method_missing(name, *args)
    "Called #{name}!"
  end
end

puts Dynamic.new.hello

Output: Called hello!

Lesson 36: Singleton Methods

Singleton Methods

Singleton methods are defined on a single object, not its class, using `def obj.method`. They enable per-object behavior customization. Singleton methods showcase Ruby’s flexibility.

obj = Object.new
def obj.greet
  "Hi!"
end

puts obj.greet

Output: Hi!

Lesson 37: Threads

Threads

Ruby’s `Thread` class creates concurrent threads, though limited by the Global Interpreter Lock (GIL). They’re useful for I/O-bound tasks. Threads require careful synchronization in Ruby.

thread = Thread.new { puts "Running" }
thread.join

Output: Running

Lesson 38: Mutex

Mutex

The `Mutex` class in Ruby’s `thread` library synchronizes access to shared resources. It prevents race conditions with `synchronize`. Mutexes are essential for thread-safe code.

require "thread"

mutex = Mutex.new
counter = 0
threads = 2.times.map do
  Thread.new do
    mutex.synchronize { counter += 1 }
  end
end
threads.each(&:join)
puts counter

Output: 2

Lesson 39: JSON Parsing

JSON Parsing

Ruby’s `json` library parses and generates JSON with `JSON.parse` and `JSON.generate`. It handles hashes and arrays seamlessly. JSON parsing is common for APIs and data exchange.

require "json"

data = { "name" => "Alice" }
json = JSON.generate(data)
puts JSON.parse(json)["name"]

Output: Alice

Lesson 40: Gems

Gems

Ruby gems are libraries installed with the `gem` command or `Gemfile` using Bundler. They extend Ruby’s functionality, like `rails` for web development. Gems are central to Ruby’s ecosystem.

# Gemfile
# source "https://rubygems.org"
# gem "faker"

require "faker"
puts Faker::Name.name

Output: (Random name, e.g., John Doe)

Lesson 41: Testing with Minitest

Testing with Minitest

Minitest is Ruby’s built-in testing framework, supporting unit tests with assertions. Tests are written in classes inheriting from `Minitest::Test`. Testing ensures code reliability in Ruby.

require "minitest/autorun"

class TestMath < Minitest::Test
  def test_add
    assert_equal 4, 2 + 2
  end
end

Output: (Test passes, no console output)

Lesson 42: Testing with RSpec

Testing with RSpec

RSpec is a popular Ruby testing gem with a BDD-style syntax. It uses `describe` and `it` for readable tests. RSpec is widely used in Ruby projects.

# Requires gem "rspec"
require "rspec"

describe "Math" do
  it "adds numbers" do
    expect(2 + 2).to eq(4)
  end
end

Output: (Test passes, no console output)

Lesson 43: Sinatra

Sinatra

Sinatra is a lightweight Ruby web framework for building HTTP servers. It uses routes like `get` to handle requests. Sinatra is ideal for small web applications.

# Requires gem "sinatra"
require "sinatra"

get "/" do
  "Hello, World!"
end

Output: (Visit http://localhost:4567 to see "Hello, World!")

Lesson 44: Sinatra Routes

Sinatra Routes

Sinatra routes map HTTP methods and paths to blocks, supporting parameters. Dynamic routes use `/:param` syntax. Routes make Sinatra flexible for web development.

# Requires gem "sinatra"
require "sinatra"

get "/hello/:name" do
  "Hello, #{params[:name]}!"
end

Output: (Visit http://localhost:4567/hello/Alice to see "Hello, Alice!")

Lesson 45: ActiveRecord

ActiveRecord

ActiveRecord, part of Rails, is an ORM for database interactions. It maps tables to classes and rows to objects. ActiveRecord simplifies database operations in Ruby.

# Requires gem "activerecord", "sqlite3"
require "active_record"

ActiveRecord::Base.establish_connection(adapter: "sqlite3", database: ":memory:")
class User < ActiveRecord::Base
end

puts "ActiveRecord connected"

Output: ActiveRecord connected

Lesson 46: ActiveRecord Models

ActiveRecord Models

ActiveRecord models represent database tables, with methods like `create` and `find`. They support validations and associations. Models are central to database-driven Ruby apps.

# Requires gem "activerecord", "sqlite3"
require "active_record"

ActiveRecord::Base.establish_connection(adapter: "sqlite3", database: ":memory:")
class User < ActiveRecord::Base
end
User.create(name: "Alice")
puts User.first.name

Output: Alice

Lesson 47: Rails Introduction

Rails Introduction

Ruby on Rails is a full-stack web framework built on Ruby, emphasizing conventions. It uses MVC (Model-View-Controller) for structure. Rails is powerful for rapid web development.

# Requires gem "rails"
# Typically in a Rails app: rails new myapp
puts "Run `rails new myapp` to start"

Output: Run `rails new myapp` to start

Lesson 48: Rails Controllers

Rails Controllers

Rails controllers handle HTTP requests, rendering views or returning data. They’re defined in `app/controllers`. Controllers are key to Rails’ MVC architecture.

# app/controllers/welcome_controller.rb
class WelcomeController < ApplicationController
  def index
    render plain: "Hello, Rails!"
  end
end

Output: (Visit http://localhost:3000/welcome to see "Hello, Rails!")

Lesson 49: Rails Routes

Rails Routes

Rails routes map URLs to controller actions, defined in `config/routes.rb`. They support RESTful resources and custom paths. Routes direct traffic in Rails applications.

# config/routes.rb
Rails.application.routes.draw do
  get "/hello", to: "welcome#index"
end

Output: (Visit http://localhost:3000/hello to see controller output)

Lesson 50: Rails Views

Rails Views

Rails views render HTML, typically using ERB templates in `app/views`. They access controller instance variables for dynamic content. Views complete the MVC cycle in Rails.

<%-- app/views/welcome/index.html.erb --%>

    @name = "World"
  end

Hello, World!
