Lesson 1: What is R?

Introduction to R

R is a programming language designed for statistical computing and graphics, widely used in data science and research. It offers extensive packages like ggplot2 and dplyr for data manipulation and visualization. R’s interactive environment makes it ideal for exploratory data analysis.

cat("Hello, World!\n")

Output: Hello, World!

Lesson 2: Variables and Assignment

Variables and Assignment

R uses `<-` or `=` for variable assignment, with `<-` being the conventional choice. Variables are dynamically typed, holding any data type. Understanding assignment is key to working with data in R.

name <- "Alice"
age <- 25
cat(name, "is", age, "\n")

Output: Alice is 25

Lesson 3: Basic Data Types

R Data Types

R supports data types like numeric, character, logical, and integer. The `typeof()` function identifies a variable’s type. Mastering data types is essential for data manipulation in R.

num <- 42
pi <- 3.14
flag <- TRUE
cat(num, pi, flag, "\n")

Output: 42 3.14 TRUE

Lesson 4: Arithmetic Operators

Arithmetic Operations

R provides arithmetic operators like `+`, `-`, `*`, `/`, and `%%` for modulus. Operations are vectorized, applying element-wise to vectors. Arithmetic is fundamental for numerical computations in R.

a <- 10
b <- 3
cat(a + b, a %% b, "\n")

Output: 13 1

Lesson 5: Comparison Operators

Comparison Operators

R’s comparison operators (`==`, `!=`, `>`, `<`, `>=`, `<=`) return logical values. They work on vectors, producing element-wise results. Comparisons drive conditional logic in R.

x <- 5
y <- 10
cat(x < y, x == y, "\n")

Output: TRUE FALSE

Lesson 6: Logical Operators

Logical Operators

R’s logical operators `&`, `|`, and `!` combine or invert logical expressions. Element-wise operators `&&` and `||` evaluate single values. Logical operators are crucial for complex conditions.

a <- TRUE
b <- FALSE
cat(a & b, a | b, "\n")

Output: FALSE TRUE

Lesson 7: Strings

Working with Strings

Strings in R are character vectors, created with quotes. Functions like `paste()` concatenate strings. Strings are central to text processing in R.

greeting <- "Hello"
name <- "World"
cat(paste(greeting, name, sep = ", "), "!\n")

Output: Hello, World!

Lesson 8: String Functions

String Functions

R provides string functions like `toupper()`, `substr()`, and `gsub()` for manipulation. These functions are vectorized for efficiency. String functions enable robust text processing in R.

text <- "  r programming  "
cat(toupper(trimws(text)), "\n")

Output: R PROGRAMMING

Lesson 9: Vectors

Working with Vectors

Vectors are R’s fundamental data structure, created with `c()` or `vector()`. They store elements of the same type. Vectors are essential for data analysis in R.

fruits <- c("apple", "banana", "cherry")
cat(fruits[2], "\n")

Output: banana

Lesson 10: Matrices

Matrices

Matrices in R are two-dimensional vectors, created with `matrix()`. They support operations like `t()` for transpose. Matrices are useful for numerical computations in R.

mat <- matrix(1:4, nrow = 2)
cat(mat[2, 1], "\n")

Output: 3

Lesson 11: If Statements

Conditional Statements

R’s `if` statements evaluate conditions, with optional `else` clauses. They return single values or vectors. If statements are key for decision-making in R.

age <- 18
if (age >= 18) {
    cat("Adult\n")
} else {
    cat("Minor\n")
}

Output: Adult

Lesson 12: Ifelse Function

Ifelse Function

R’s `ifelse()` function applies conditional logic element-wise to vectors. It’s more concise than `if` for vectorized operations. Ifelse is widely used in data transformations.

ages <- c(15, 20)
result <- ifelse(ages >= 18, "Adult", "Minor")
cat(result, "\n")

Output: Minor Adult

Lesson 13: For Loops

For Loops

R’s `for` loops iterate over vectors or sequences using `in`. They’re useful for explicit iteration, though vectorization is preferred. For loops are common in R scripting.

for (i in 0:2) {
    cat(i, "\n")
}

Output:
0
1
2

Lesson 14: While Loops

While Loops

R’s `while` loops execute while a condition is true. They’re less common due to vectorization but useful for dynamic iteration. While loops handle repetitive tasks in R.

i <- 0
while (i < 3) {
    cat(i, "\n")
    i <- i + 1
}

Output:
0
1
2

Lesson 15: Break and Next

Loop Control

R’s `break` exits a loop, while `next` skips to the next iteration. They work in `for` and `while` loops. Loop control statements manage iteration flow in R.

for (i in 0:4) {
    if (i == 3) break
    cat(i, "\n")
}

Output:
0
1
2

Lesson 16: Functions

Defining Functions

Functions in R are defined with `function()`, supporting default arguments. They return the last evaluated expression. Functions promote reusable code in R.

greet <- function(name) {
    paste("Hello,", name, "!")
}
cat(greet("Alice"), "\n")

Output: Hello, Alice!

Lesson 17: Anonymous Functions

Anonymous Functions

R supports anonymous functions, often used in `lapply()` or `sapply()`. They’re defined inline without a name. Anonymous functions are common in functional programming in R.

numbers <- c(1, 2, 3)
result <- lapply(numbers, function(x) x * 2)
cat(unlist(result), "\n")

Output: 2 4 6

Lesson 18: Data Frames

Data Frames

Data frames in R are tabular structures, like spreadsheets, created with `data.frame()`. They store heterogeneous data with named columns. Data frames are central to data analysis in R.

df <- data.frame(name = c("Alice", "Bob"), age = c(25, 30))
cat(df$name[1], "\n")

Output: Alice

Lesson 19: Lists

Lists

Lists in R are ordered collections of mixed types, created with `list()`. They’re accessed with `[[ ]]` for single elements or `$` for named items. Lists are versatile for complex data in R.

person <- list(name = "Alice", age = 25)
cat(person$name, "\n")

Output: Alice

Lesson 20: Factors

Factors

Factors in R represent categorical data, created with `factor()`. They store levels for efficient analysis. Factors are crucial for statistical modeling in R.

gender <- factor(c("Male", "Female", "Male"))
cat(levels(gender), "\n")

Output: Female Male

Lesson 21: Basic Plotting

Basic Plotting

R’s base plotting system creates visualizations with `plot()`. It supports scatter plots, lines, and customization. Plotting is a core strength of R for data exploration.

Lesson 22: Bar Plots

Bar Plots

R’s `barplot()` creates bar charts for categorical data. It supports customization like colors and labels. Bar plots visualize frequency or summary data in R.

counts <- c(5, 10, 15)
barplot(counts, names.arg = c("A", "B", "C"))

Output: (Displays a bar chart with three bars)

Lesson 23: Apply Functions

Apply Functions

R’s `apply()` family (`lapply`, `sapply`, `tapply`) iterates over data structures, avoiding loops. They’re vectorized for performance. Apply functions are idiomatic for R programming.

mat <- matrix(1:4, nrow = 2)
row_sums <- apply(mat, 1, sum)
cat(row_sums, "\n")

Lesson 24: Missing Values

Missing Values

R uses `NA` for missing values, handled with `is.na()` and `na.omit()`. Missing data affects computations, requiring careful handling. Managing `NA` is critical for data analysis in R.

data <- c(1, NA, 3)
cat(sum(data, na.rm = TRUE), "\n")

Output: 4

Lesson 25: Subsetting

Subsetting

R’s subsetting uses `[ ]`, `[[ ]]`, or `$` for vectors, lists, and data frames. Logical indexing filters data efficiently. Subsetting is key for data manipulation in R.

data <- c(10, 20, 30)
cat(data[data > 15], "\n")

Output: 20 30

Lesson 26: Data Frame Subsetting

Data Frame Subsetting

Data frames in R are subset using `[ ]` with row and column indices or names. Logical conditions filter rows effectively. Subsetting data frames is essential for analysis in R.

df <- data.frame(name = c("Alice", "Bob"), age = c(25, 30))
cat(df[df$age > 25, "name"], "\n")

Output: Bob

Lesson 27: Merging Data Frames

Merging Data Frames

R’s `merge()` or `dplyr::join()` combines data frames by common columns. It supports inner, left, right, and full joins. Merging is crucial for integrating datasets in R.

df1 <- data.frame(id = 1:2, name = c("Alice", "Bob"))
df2 <- data.frame(id = 1:2, age = c(25, 30))
merged <- merge(df1, df2, by = "id")
cat(merged$name[1], merged$age[1], "\n")

Output: Alice 25

Lesson 28: Dplyr Introduction

Dplyr Introduction

R’s `dplyr` package provides intuitive functions like `filter()`, `select()`, and `mutate()` for data manipulation. It uses a pipe operator (`%>%`) for chaining. Dplyr is a cornerstone of modern R data analysis.

library(dplyr)
df <- data.frame(name = c("Alice", "Bob"), age = c(25, 30))
result <- df %>% filter(age > 25)
cat(result$name, "\n")

Output: Bob

Lesson 29: Dplyr Grouping

Dplyr Grouping

Dplyr’s `group_by()` and `summarise()` aggregate data by groups. They compute summaries like means or counts. Grouping is powerful for summarizing data in R.

library(dplyr)
df <- data.frame(group = c("A", "A", "B"), value = c(10, 20, 30))
result <- df %>% group_by(group) %>% summarise(mean_val = mean(value))
cat(result$mean_val, "\n")

Output: 15 30

Lesson 30: Tidyr Pivoting

Tidyr Pivoting

R’s `tidyr` package reshapes data with `pivot_longer()` and `pivot_wider()`. Pivoting transforms data between wide and long formats. Tidyr is essential for tidy data in R.

library(tidyr)
df <- data.frame(id = 1, A = 10, B = 20)
long <- df %>% pivot_longer(cols = c(A, B), names_to = "category", values_to = "value")
cat(long$category[1], long$value[1], "\n")

Output: A 10

Lesson 39: Linear Regression

Linear Regression

R’s `lm()` fits linear models, with `summary()` providing coefficients and diagnostics. It’s used for predictive modeling. Linear regression is a core statistical tool in R.

Lesson 41: Machine Learning with Caret

Machine Learning with Caret

R’s `caret` package streamlines machine learning with functions like `train()`. It supports models like random forests. Caret simplifies model training and evaluation in R.

library(caret)
df <- data.frame(x = 1:10, y = (1:10) + rnorm(10))
model <- train(y ~ x, data = df, method = "lm")
cat(predict(model, df)[1], "\n")

Output: (Sample prediction, e.g., 1.2)

Lesson 42: Cross-Validation

Cross-Validation

Caret’s `trainControl()` enables k-fold cross-validation for robust model evaluation. It reduces overfitting by splitting data. Cross-validation is key for machine learning in R.

library(caret)
df <- data.frame(x = 1:10, y = (1:10) + rnorm(10))
ctrl <- trainControl(method = "cv", number = 5)
model <- train(y ~ x, data = df, method = "lm", trControl = ctrl)
cat(model$results$RMSE, "\n")

Output: (Sample RMSE, e.g., 1.1)

Lesson 43: Shiny Introduction

Shiny Introduction

R’s `shiny` package builds interactive web applications with reactive UI and server logic. It uses `ui.R` and `server.R` or a single app file. Shiny is ideal for data-driven web apps in R.

Lesson 45: Shiny Plots

Shiny Plots

Shiny integrates ggplot2 plots with `renderPlot()` for dynamic visualizations. Inputs control plot parameters reactively. Plots enhance Shiny’s data exploration capabilities in R.

library(shiny)
library(ggplot2)
ui <- fluidPage(sliderInput("n", "Points", 1, 10, 5), plotOutput("plot"))
server <- function(input, output) {
    output$plot <- renderPlot({
        ggplot(data.frame(x = 1:input$n, y = (1:input$n)^2), aes(x, y)) + geom_point()
    })
}
# shinyApp(ui, server)

Output: (Displays a scatter plot based on slider input)

Lesson 50: Parallel Computing

Parallel Computing

R’s `parallel` package enables parallel processing with `parLapply()`. It speeds up computations on multicore systems. Parallel computing optimizes performance for large datasets in R.

library(parallel)
cl <- makeCluster(2)
result <- parLapply(cl, 1:2, function(x) x^2)
stopCluster(cl)
cat(unlist(result), "\n")

Output: 1 4
