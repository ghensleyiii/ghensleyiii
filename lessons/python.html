Lesson 1: What is Python?

Introduction to Python

Python is a high-level, interpreted programming language known for its readability and versatility. It supports multiple paradigms, including procedural, object-oriented, and functional programming. Python is widely used in web development, data science, and automation.

print("Hello, World!")

Output: Hello, World!

Lesson 2: Variables

Variables in Python

Variables store data values and are created by assigning a value with the equals sign. Python is dynamically typed, so you don’t need to declare the variable type. Variables are essential for manipulating and storing data in programs.

name = "Alice"
age = 25
print(name, age)

Output: Alice 25

Lesson 3: Data Types

Basic Data Types

Python supports various data types, including integers, floats, strings, and booleans. The type() function reveals a variable’s data type. Understanding data types is crucial for performing appropriate operations.

x = 5        # int
y = 3.14     # float
z = "Hello"  # str
print(type(x), type(y), type(z))

Lesson 4: Arithmetic Operators

Arithmetic Operations

Python provides operators like +, -, *, /, and ** for arithmetic calculations. The // operator performs floor division, and % returns the remainder. These operators are fundamental for numerical computations.

a = 10
b = 3
print(a + b, a * b, a // b, a % b)

Output: 13 30 3 1

Lesson 5: Comparison Operators

Comparison Operators

Comparison operators like ==, !=, >, and < compare values and return booleans. They are used in conditional statements to control program flow. Understanding comparisons is key to decision-making in code.

x = 5
y = 10
print(x == y, x < y, x >= y)

Output: False True False

Lesson 6: Logical Operators

Logical Operators

Logical operators and, or, and not combine or invert boolean expressions. They are used to create complex conditions in control structures. Logical operators help evaluate multiple conditions efficiently.

a = True
b = False
print(a and b, a or b, not a)

Output: False True False

Lesson 7: Strings

Working with Strings

Strings are sequences of characters enclosed in single or double quotes. They support operations like concatenation (+) and repetition (*). Strings are fundamental for text processing in Python.

greeting = "Hello"
name = "World"
print(greeting + ", " + name + "!")

Output: Hello, World!

Lesson 8: String Methods

String Methods

Python strings have built-in methods like upper(), lower(), and strip() for manipulation. These methods return new strings without modifying the original. String methods simplify text processing tasks.

text = "  Python  "
print(text.strip().upper())

Lesson 9: String Formatting

String Formatting

Python offers f-strings, .format(), and % for string formatting. F-strings, introduced in Python 3.6, are concise and readable. Formatting is essential for dynamic text output.

name = "Alice"
age = 25
print(f"{name} is {age} years old.")

Output: Alice is 25 years old.

Lesson 10: Input Function

User Input

The input() function captures user input as a string. You can convert it to other types, like int, for numerical operations. Input enables interactive programs.

Lesson 11: If Statements

Conditional Statements

If statements execute code based on conditions using if, elif, and else. They allow programs to make decisions dynamically. Conditionals are the backbone of program logic.

age = 18
if age >= 18:
    print("Adult")
else:
    print("Minor")

Output: Adult

Lesson 12: Nested If Statements

Nested Conditionals

Nested if statements place conditions inside other if blocks for complex logic. They allow detailed decision trees but should be kept simple for readability. Nesting is useful for multi-level checks.

score = 85
if score >= 60:
    if score >= 80:
        print("Excellent")
    else:
        print("Pass")
else:
    print("Fail")

Output: Excellent

Lesson 13: While Loops

While Loops

While loops repeat code as long as a condition is true. They’re useful for tasks with unknown iteration counts. Use break to exit loops early.

count = 1
while count <= 3:
    print(count)
    count += 1

Output:
1
2
3

Lesson 14: For Loops

For Loops

For loops iterate over sequences like lists or ranges. They’re ideal for known iteration counts. The range() function generates number sequences for looping.

for i in range(3):
    print(i)

Output:
0
1
2

Lesson 15: Break and Continue

Loop Control

The break statement exits a loop immediately, while continue skips to the next iteration. These statements provide fine-grained control over loop behavior. They’re useful for handling special cases in loops.

for i in range(5):
    if i == 3:
        break
    print(i)

Output:
0
1
2

Lesson 16: Functions

Defining Functions

Functions, defined with def, encapsulate reusable code blocks. They accept parameters and can return values. Functions promote code modularity and reusability.

def greet(name):
    return f"Hello, {name}!"
print(greet("Alice"))

Output: Hello, Alice!

Lesson 17: Default Parameters

Default Parameters

Functions can have default parameter values, used when arguments are omitted. Default parameters simplify function calls and enhance flexibility. They’re useful for optional settings.

def greet(name="Guest"):
    return f"Hello, {name}!"
print(greet())
print(greet("Bob"))

Output:
Hello, Guest!
Hello, Bob!

Lesson 18: Keyword Arguments

Keyword Arguments

Keyword arguments allow passing arguments by name, improving readability. They enable flexible argument order in function calls. Keyword arguments are useful for functions with many parameters.

def describe_person(name, age):
    return f"{name} is {age}."
print(describe_person(age=30, name="Alice"))

Working with Lists

Lists are mutable, ordered collections defined with square brackets. They support indexing, slicing, and methods like append(). Lists are versatile for storing and manipulating data.

Lesson 20: List Slicing

List Slicing

Lesson 21: Tuples

Working with Tuples

Tuples are immutable, ordered collections defined with parentheses. They’re used for fixed data that shouldn’t change. Tuples are faster than lists for certain operations.

Lesson 22: Dictionaries

Dictionaries

Dictionaries store key-value pairs, defined with curly braces. They allow fast lookups by key and are mutable. Dictionaries are ideal for structured data.

Lesson 23: Sets

Working with Sets

Sets are unordered, mutable collections of unique elements, defined with curly braces or set(). They support operations like union and intersection. Sets are useful for eliminating duplicates and performing mathematical operations.

Lesson 24: List Methods

List Methods

Lists have methods like remove(), pop(), and sort() for manipulation. These methods modify lists in place or return values. List methods streamline common operations.

numbers.sort()

Lesson 25: Dictionary Methods

Dictionary Methods

Dictionaries offer methods like get(), keys(), and values() for accessing data. The get() method safely retrieves values with a default if the key is missing. These methods enhance dictionary usability.

person = {"name": "Bob", "age": 30}

Set Operations

Sets support operations like union (|), intersection (&), and difference (-). These operations are efficient for comparing collections. Set operations are useful for data analysis tasks.

Lesson 33: Classes

Object-Oriented Programming

Classes define blueprints for objects, encapsulating data and behavior. They use methods and attributes to model real-world entities. Classes are central to object-oriented programming in Python.

class Dog:
    def __init__(self, name):
        self.name = name
    def bark(self):
        return f"{self.name} says Woof!"
dog = Dog("Buddy")
print(dog.bark())

Output: Buddy says Woof!

Lesson 34: Inheritance

Class Inheritance

Inheritance allows a class to inherit attributes and methods from another. It promotes code reuse and hierarchical modeling. Inheritance is powerful for extending functionality.

class Animal:
    def __init__(self, name):
        self.name = name
class Dog(Animal):
    def bark(self):
        return f"{self.name} barks!"
dog = Dog("Max")
print(dog.bark())

Output: Max barks!

Lesson 35: List Comprehensions

List Comprehensions

List comprehensions provide a concise way to create lists from iterables. They combine loops and conditionals in one line. Comprehensions improve code readability and efficiency.

squares = [x**2 for x in range(5)]
print(squares)

Output: [0, 1, 4, 9, 16]

Lesson 36: Dictionary Comprehensions

Dictionary Comprehensions

Dictionary comprehensions create dictionaries from iterables in a concise syntax. They’re useful for transforming or filtering data. They mirror list comprehensions for key-value pairs.

squares = {x: x**2 for x in range(4)}
print(squares)

Output: {0: 0, 1: 1, 2: 4, 3: 9}

Lesson 37: Lambda Functions

Lambda Functions

Lambda functions are anonymous, single-expression functions defined with lambda. They’re used for short, throwaway functions, often in higher-order functions. Lambda functions streamline functional programming.

double = lambda x: x * 2
print(double(5))

Output: 10

Lesson 38: Map and Filter

Map and Filter Functions

The map() function applies a function to all items in an iterable, while filter() selects items based on a condition. Both return iterators, often used with lambdas. They’re powerful for functional-style programming.

numbers = [1, 2, 3]
squares = list(map(lambda x: x**2, numbers))
print(squares)

Output: [1, 4, 9]

Lesson 39: Generators

Generators

Generators yield values one at a time, using yield, conserving memory for large datasets. They’re defined like functions but behave like iterators. Generators are ideal for lazy evaluation.

def count_up_to(n):
    for i in range(n):
        yield i
gen = count_up_to(3)
print(list(gen))

Output: [0, 1, 2]

Lesson 40: Decorators

Decorators

Decorators wrap functions to modify their behavior, using the @ syntax. They’re useful for logging, timing, or access control. Decorators enhance code modularity without altering function code.

def my_decorator(func):
    def wrapper():
        print("Before")
        func()
        print("After")
    return wrapper

@my_decorator
def say_hello():
    print("Hello!")
say_hello()

Output:
Before
Hello!
After

Lesson 41: Regular Expressions

Regular Expressions

The re module provides regular expressions for pattern matching in strings. Functions like re.search() and re.findall() extract or validate data. Regex is powerful for text processing.

import re
text = "Contact: 123-456-7890"
match = re.search(r"\d{3}-\d{3}-\d{4}", text)
print(match.group())

Output: 123-456-7890

Lesson 42: Property Decorators

Property Decorators

The @property decorator creates getter methods for class attributes. It allows controlled access to private attributes. Properties simplify attribute management in classes.

class Person:
    def __init__(self, name):
        self._name = name
    @property
    def name(self):
        return self._name
p = Person("Alice")
print(p.name)

Output: Alice

Lesson 43: Static Methods

Static Methods

Static methods, defined with @staticmethod, don’t require self or cls parameters. They’re utility functions within a class namespace. Static methods organize related functionality without instance dependence.

class MathUtils:
    @staticmethod
    def add(a, b):
        return a + b
print(MathUtils.add(5, 3))

Output: 8

Lesson 44: Class Methods

Class Methods

Class methods, defined with @classmethod, take cls as their first parameter. They operate on the class itself, not instances. Class methods are useful for factory methods or class-level operations.

class MyClass:
    @classmethod
    def from_string(cls, string):
        return cls(string)
    def __init__(self, value):
        self.value = value
obj = MyClass.from_string("Test")
print(obj.value)

Output: Test

Lesson 45: Context Managers

Context Managers

Context managers, used with the with statement, manage resources like files or connections. They define __enter__ and __exit__ methods for setup and cleanup. Context managers ensure proper resource handling.

class MyContext:
    def __enter__(self):
        print("Entering")
        return self
    def __exit__(self, *args):
        print("Exiting")
with MyContext():
    print("Inside")

Output:
Entering
Inside
Exiting

Lesson 49: Datetime

Datetime Module

The datetime module handles dates and times, with classes like datetime and date. It supports formatting and arithmetic operations. Datetime is essential for time-based applications.

from datetime import datetime
now = datetime.now()

Lesson 50: Args and Kwargs

Variable Arguments

*args collects positional arguments into a tuple, while **kwargs collects keyword arguments into a dictionary. They allow functions to accept variable numbers of arguments. Args and kwargs enhance function flexibility.

def func(*args, **kwargs):
    print(args, kwargs)
func(1, 2, a=3, b=4)

Output: (1, 2) {'a': 3, 'b': 4}
