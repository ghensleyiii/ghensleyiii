Lesson 1: What is PHP?

Introduction to PHP

PHP is a server-side scripting language designed for web development, embedded in HTML. It powers platforms like WordPress and is known for its simplicity and database integration. PHP’s versatility makes it ideal for dynamic websites and APIs.

<?php
echo "Hello, World!";
?>

Output: Hello, World!

Lesson 2: Variables

Variables in PHP

Variables in PHP are declared with `$` and are dynamically typed, requiring no type definition. They hold various data types, assigned by value or reference. PHP’s loose typing simplifies coding but demands careful validation.

<?php
$name = "Alice";
$age = 25;
echo "$name is $age";
?>

Output: Alice is 25

Lesson 3: Data Types

PHP Data Types

PHP supports scalar types (int, float, string, bool), compound types (array, object), and special types (null, resource). Types are dynamically assigned, with automatic conversion in some contexts. Understanding data types is crucial for effective PHP programming.

<?php
$num = 42;
$pi = 3.14;
$flag = true;
var_dump($num, $pi, $flag);
?>

Output:
int(42)
float(3.14)
bool(true)

Lesson 4: Arithmetic Operators

Arithmetic Operations

PHP provides arithmetic operators like `+`, `-`, `*`, `/`, `%`, and `**` for exponentiation. They work on numeric types, with automatic type conversion. These operators are fundamental for calculations in PHP.

<?php
$a = 10;
$b = 3;
echo $a + $b, "\n", $a % $b;
?>

Output:
13
1

Lesson 5: Comparison Operators

Comparison Operators

PHP’s comparison operators (`==`, `===`, `!=`, `!==`, `>`, `<`) return booleans. The `===` checks value and type, while `==` allows type coercion. Comparisons are essential for conditional logic.

<?php
$x = 5;
$y = "5";
var_dump($x == $y, $x === $y);
?>

Output:
bool(true)
bool(false)

Lesson 6: Logical Operators

Logical Operators

PHP’s logical operators `&&`, `||`, `!`, `and`, `or` combine or invert boolean expressions. They short-circuit for efficiency, evaluating only as needed. Logical operators are key for complex conditions.

<?php
$a = true;
$b = false;
var_dump($a && $b, $a || $b);
?>

Output:
bool(false)
bool(true)

Lesson 7: Strings

Working with Strings

Strings in PHP are sequences of characters, defined with single or double quotes. Double-quoted strings support variable interpolation with `{}` or direct `$var`. Strings are central to web output and data processing.

<?php
$greeting = "Hello";
$name = "World";
echo "$greeting, $name!";
?>

Output: Hello, World!

Lesson 8: String Functions

String Functions

PHP offers functions like `strlen`, `strtoupper`, and `str_replace` for string manipulation. These functions are efficient and widely used in web applications. String functions simplify text processing tasks.

<?php
$text = "  php  ";
echo trim(strtoupper($text));
?>

Output: PHP

Lesson 9: Arrays

Working with Arrays

Arrays in PHP are ordered maps, created with `[]` or `array()`. They support indexed and associative forms, with dynamic resizing. Arrays are versatile for storing and manipulating data.

<?php
$fruits = ["apple", "banana", "cherry"];
echo $fruits[1];
?>

Output: banana

Lesson 10: Associative Arrays

Associative Arrays

Associative arrays in PHP use string or integer keys, defined with `=>`. They’re ideal for key-value data, like configurations. Associative arrays are a core feature of PHP.

<?php
$ages = ["Alice" => 25, "Bob" => 30];
echo $ages["Alice"];
?>

Output: 25

Lesson 11: If Statements

Conditional Statements

PHP’s `if` statements evaluate conditions, supporting `elseif` and `else`. They use curly braces or alternative syntax for templates. Conditionals are essential for dynamic web logic.

<?php
$age = 18;
if ($age >= 18) {
    echo "Adult";
} else {
    echo "Minor";
}
?>

Output: Adult

Lesson 12: Switch Statements

Switch Statements

PHP’s `switch` statement matches a value against cases, with explicit `break` to prevent fall-through. It supports a `default` case for unmatched values. Switch is useful for multi-way branching.

<?php
$day = 1;
switch ($day) {
    case 1:
        echo "Monday";
        break;
    default:
        echo "Other";
}
?>

Output: Monday

Lesson 13: For Loops

For Loops

PHP’s `for` loop iterates with a counter, using initialization, condition, and increment expressions. It’s ideal for known iteration counts. For loops are widely used in PHP scripting.

<?php
for ($i = 0; $i < 3; $i++) {
    echo $i . "\n";
}
?>

Output:
0
1
2

Lesson 14: Foreach Loops

Foreach Loops

The `foreach` loop iterates over arrays, yielding keys and values. It supports both indexed and associative arrays. Foreach is idiomatic for array processing in PHP.

<?php
$fruits = ["apple", "banana"];
foreach ($fruits as $fruit) {
    echo $fruit . "\n";
}
?>

Output:
apple
banana

Lesson 15: Break and Continue

Loop Control

The `break` statement exits a loop, while `continue` skips to the next iteration. Both can take a number to affect nested loops. These controls manage loop execution in PHP.

<?php
for ($i = 0; $i < 5; $i++) {
    if ($i == 3) break;
    echo $i . "\n";
}
?>

Output:
0
1
2

Lesson 16: Functions

Defining Functions

Functions in PHP are declared with `function`, supporting parameters and return values. They can have default arguments and type hints. Functions promote reusable code in PHP.

<?php
function greet($name) {
    return "Hello, $name!";
}
echo greet("Alice");
?>

Output: Hello, Alice!

Lesson 17: Type Hints

Type Hints

PHP supports type hints for function parameters and return types, enforcing types at runtime. They include scalar types, classes, and `void`. Type hints improve code reliability.

<?php
function add(int $a, int $b): int {
    return $a + $b;
}
echo add(2, 3);
?>

Output: 5

Lesson 18: Classes

Classes

Classes in PHP define objects with properties and methods, declared with `class`. They support visibility modifiers like `public`, `private`. Classes are key to object-oriented PHP.

<?php
class Dog {
    public $name;
    public function __construct($name) {
        $this->name = $name;
    }
    public function bark() {
        return "$this->name says Woof!";
    }
}
$dog = new Dog("Buddy");
echo $dog->bark();
?>

Output: Buddy says Woof!

Lesson 19: Inheritance

Inheritance

PHP supports single inheritance with `extends`, allowing subclasses to inherit properties and methods. The `parent::` syntax calls parent methods. Inheritance organizes related classes hierarchically.

<?php
class Animal {
    public function speak() {
        return "Sound";
    }
}
class Dog extends Animal {
    public function speak() {
        return parent::speak() . ": Woof!";
    }
}
$dog = new Dog();
echo $dog->speak();
?>

Output: Sound: Woof!

Lesson 20: Interfaces

Interfaces

Interfaces in PHP define method signatures that classes must implement, using `implements`. They enable polymorphism without inheritance. Interfaces are crucial for flexible designs.

<?php
interface Speaker {
    public function speak();
}
class Dog implements Speaker {
    public function speak() {
        return "Woof!";
    }
}
$dog = new Dog();
echo $dog->speak();
?>

Output: Woof!

Lesson 21: Error Handling

Error Handling

PHP uses `try`, `catch`, and `throw` for exception handling, with `Exception` as the base class. Exceptions are caught by type, and `finally` ensures cleanup. Error handling ensures robust applications.

<?php
try {
    throw new Exception("Error!");
} catch (Exception $e) {
    echo "Caught: " . $e->getMessage();
}
?>

Output: Caught: Error!

Lesson 22: Custom Exceptions

Custom Exceptions

Custom exceptions in PHP are created by extending `Exception`. They provide specific error context for better debugging. Custom exceptions enhance error handling clarity.

<?php
class MyException extends Exception {}
try {
    throw new MyException("Custom error");
} catch (MyException $e) {
    echo $e->getMessage();
}
?>

Output: Custom error

Lesson 23: Superglobals

Superglobals

PHP’s superglobals like `$_GET`, `$_POST`, and `$_SERVER` provide access to request data and server info. They’re globally available in any scope. Superglobals are essential for web interactions.

<?php

$_GET['name'] = "Alice";
echo $_GET['name'];
?>

Lesson 24: Form Handling

Form Handling

PHP handles form data via `$_POST` or `$_GET`, depending on the form’s method. Input validation prevents security issues like XSS. Form handling is a core feature of PHP web apps.

Lesson 25: Sessions

Sessions

Lesson 26: Cookies

Cookies

PHP sets cookies with `setcookie()`, storing data in the browser. Cookies are accessed via `$_COOKIE`. Cookies are used for persistent client-side data.

<?php

Lesson 27: File Writing

File Writing

PHP’s `file_put_contents()` writes data to files, with error handling for failures. It’s atomic and simple for basic I/O. File writing is crucial for persistent storage.

Lesson 28: File Reading

File Reading

The `file_get_contents()` function reads entire files into strings. It throws warnings on failure, requiring error checks. File reading is essential for data retrieval.

Lesson 29: Regular Expressions

Regular Expressions

PHP’s PCRE functions like `preg_match` and `preg_replace` handle regular expressions. Patterns are defined with delimiters (e.g., `/pattern/`). Regexes are powerful for text validation and manipulation.

<?php

Lesson 30: JSON Encoding

JSON Encoding

PHP’s `json_encode()` converts arrays and objects to JSON strings. It handles nested structures and supports options like `JSON_PRETTY_PRINT`. JSON encoding is common for APIs.

<?php
$data = ["name" => "Alice"];
echo json_encode($data);
?>

Lesson 31: JSON Decoding

JSON Decoding

The `json_decode()` function converts JSON to PHP arrays or objects. It supports an associative array option for flexibility. JSON decoding is vital for API data processing.

<?php
$json = '{"name":"Alice"}';
$data = json_decode($json, true);
echo $data["name"];
?>

Output: Alice

Lesson 32: PDO

PDO

PHP’s PDO (PHP Data Objects) provides a consistent interface for database access. It supports prepared statements for security. PDO is preferred for modern database interactions.

<?php
$pdo = new PDO("sqlite::memory:");

?>

Output: Database connected

Lesson 33: PDO Queries

PDO Queries

PDO’s `query` and `prepare` methods execute SQL, with `fetch` retrieving results. Prepared statements prevent SQL injection. PDO queries are secure and efficient for databases.

<?php
$pdo = new PDO("sqlite::memory:");
$pdo->exec("CREATE TABLE users (name TEXT)");
$stmt = $pdo->prepare("INSERT INTO users (name) VALUES (?)");
$stmt->execute(["Alice"]);
echo $pdo->query("SELECT name FROM users")->fetchColumn();
?>

Output: Alice

Lesson 34: Array Functions

Array Functions

PHP provides functions like `array_map`, `array_filter`, and `array_reduce` for array manipulation. They support callbacks for custom logic. Array functions simplify complex data processing.

<?php

$squares = array_map(fn($n) => $n * $n, $numbers);
print_r($squares);
?>

Lesson 35: Anonymous Functions

Anonymous Functions

Anonymous functions, or closures, are defined with `function` or arrow syntax `fn`. They can capture variables with `use`. Closures are useful for callbacks and functional programming.

Lesson 36: Traits

Traits

Traits in PHP provide reusable methods for classes, included with `use`. They support method precedence and aliasing. Traits enhance code reuse without inheritance.

<?php
trait Greetable {

        return "Hello!";
    }

class Person {

?>

Lesson 37: Namespaces

Namespaces

Namespaces in PHP organize code, declared with `namespace` to avoid name collisions. They’re imported with `use` or fully qualified names. Namespaces are essential for large projects.

Lesson 38: Autoloading

Autoloading

Lesson 48: Eloquent ORM

Eloquent ORM

Laravel’s Eloquent ORM maps database tables to models, defined in `app/Models`. It supports queries, relationships, and validations. Eloquent simplifies database operations.

<?php
// app/Models/User.php
namespace App\Models;
use Illuminate\Database\Eloquent\Model;
class User extends Model {}
// routes/web.php
use App\Models\User;
Route::get("/user", function () {
    $user = new User;
    $user->name = "Alice";
    $user->save();
    return $user->name;
});
?>

Output: Alice

Lesson 49: Laravel Middleware

Laravel Middleware

Laravel middleware filters HTTP requests, defined in `app/Http/Middleware`. They handle tasks like authentication or logging. Middleware enhances request processing flexibility.

<?php
// app/Http/Middleware/LogRequest.php
namespace App\Http\Middleware;
use Closure;
class LogRequest {
    public function handle($request, Closure $next) {
        logger("Request received");
        return $next($request);
    }
}
// routes/web.php: Route::get("/", fn() => "Hello!")->middleware(LogRequest::class);
?>

Output: Hello! (Logs "Request received" in logs)

Lesson 50: Laravel Authentication

Laravel Authentication

Laravel provides built-in authentication with `artisan make:auth` or `Laravel Breeze`. It handles login, registration, and password resets. Authentication is critical for secure web apps.

<?php
// routes/web.php
use Illuminate\Support\Facades\Auth;
use Illuminate\Support\Facades\Route;
Route::get("/user", function () {
    return Auth::check() ? Auth::user()->name : "Guest";
});
?>

Output: Guest (or user name if authenticated)
